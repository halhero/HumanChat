# 05 多模态对话体验

## 目标

当前项目有 TTS 语音输出，但输入仍然是文字。本阶段加入第一版语音输入能力，让项目从“文字聊天 + 语音回复”升级为“可语音输入的聊天 Agent”。

## 设计目的

- 让输入层可扩展，不再把 `input()` 写死为唯一来源。
- 第一版先支持“从音频文件转文字”或“可选录音依赖”，避免引入过重的实时音频复杂度。
- 复用现有聊天流程，不改动 Agent 核心。

## 计划代码

新增 `human_chat/input_provider.py`：

```python
class TextInputProvider:
    def read_question(self) -> str:
        return input("\n你：").strip()

class AudioFileInputProvider:
    def read_question(self) -> str:
        path = input("音频文件路径：")
        return transcribe_audio_file(path)
```

## 完成标准

- 默认文字输入仍然可用。
- 可以选择音频文件输入模式。
- STT 功能是可选依赖或 API 配置，不影响普通文字聊天。
