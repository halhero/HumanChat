# 02 对话运行层整理

## 目标

当前 `cli.py` 同时负责用户输入、会话选择、调用 LangGraph、保存 session、处理 TTS 生命周期。功能能跑，但职责偏重。本阶段要抽出运行层，让 CLI 只负责交互。

## 设计目的

- 把核心对话运行逻辑从 CLI 中分离出来。
- 为未来 Web UI、桌面 UI、语音输入复用同一套运行逻辑。
- 降低后续扩展时对 `cli.py` 的耦合。

## 计划代码

新增 `human_chat/runtime.py`：

```python
class ChatRuntime:
    def __init__(self, settings, session):
        self.settings = settings
        self.session = session
        self.app = build_graph(settings)
        self.messages = dicts_to_messages(session.get("messages", []))

    def ask(self, question: str) -> dict:
        result = self.app.invoke(...)
        self.session["messages"] = messages_to_dicts(self.messages)
        save_session(...)
        return result
```

CLI 中只保留：

```python
runtime = ChatRuntime(settings, session)
result = runtime.ask(question)
```

## 完成标准

- `cli.py` 不再直接维护 `messages` 和保存 session。
- `run_once()` 和 `chat_loop()` 可以复用运行层。
- 行为保持一致：会话仍然可恢复，聊天仍然自动保存。
