# 12 工具调用框架化修改记录

## 本次目标

本次改造对应 `资料/11-框架化改造盘点.md` 中的第一项：工具定义与工具调用框架化。

目标是把原先手写的工具决策和工具执行逻辑，迁移到 LangChain / LangGraph 的标准工具机制上。

## 修改前的问题

原先 `human_chat/graph.py` 中存在这些自定义逻辑：

```text
ToolDecision
TOOL_DECISION_PROMPT
decide_tool_use
execute_tool
route_after_tool_decision
```

这相当于自己实现了一套工具调用协议：

```text
模型输出 tool_name / arguments
代码用 if/else 判断工具名
代码手动调用工具函数
再把结果写入 tool_result 字符串
```

这种方式可以工作，但不够 LangGraph / LangChain 化。

## 本次怎么改

### 1. 工具函数增加 LangChain tool wrapper

在 `human_chat/tools.py` 中保留原来的底层 Python 函数：

```text
list_project_files
read_project_file
search_project_text
```

同时新增框架工具：

```text
list_project_files_tool
read_project_file_tool
search_project_text_tool
get_project_tools
```

这些工具使用 `@tool` 暴露给 LangChain，使模型可以基于工具名称、描述和参数 schema 调用它们。

### 2. Graph 使用 bind_tools 和 ToolNode

在 `human_chat/graph.py` 中新增：

```python
project_tools = get_project_tools()
tool_llm = llm.bind_tools(project_tools)
tool_node = ToolNode(project_tools, messages_key="tool_messages")
```

这替代了原先手写的 `ToolDecision` 和 `execute_tool`。

### 3. 使用独立 tool_messages 通道

在 `ChatState` 中新增：

```python
tool_messages: Annotated[list[Any], add_messages]
```

工具调用消息只保存在本轮 Graph 的 scratch 通道中，不进入最终会话历史。

这么做的目的是：

```text
使用 LangGraph ToolNode
避免 session messages 被内部工具调用污染
最终会话仍只保存用户消息和助手回复
```

### 4. 删除自定义 ToolDecision

`human_chat/schemas.py` 中移除了：

```text
ToolDecision
tool_request
tool_result
```

工具调用结果改由 `ToolMessage` 承载，并在最终回复前格式化进系统上下文。

## 修改后的流程

当前 Graph 流程变为：

```text
prepare_context
  -> call_project_tools_model
  -> execute_project_tools 或 generate_reply
  -> generate_reply
  -> synthesize_speech
```

其中：

```text
call_project_tools_model：绑定工具的模型判断是否需要工具
execute_project_tools：LangGraph ToolNode 执行工具
generate_reply：读取工具结果并生成最终结构化回复
```

## 为什么这样更好

这次改造带来几个好处：

```text
工具定义符合 LangChain 标准
工具执行交给 LangGraph ToolNode
新增工具时不需要改 graph.py 的 if/else
后续接 MCP 工具更自然
保留 CLI 手动工具命令，不破坏现有使用方式
```

## 后续注意

当前仍然有一些后续优化空间：

```text
工具调用消息目前是本轮 scratch 状态，暂不持久化
后续接入 checkpointer 后，需要确认 tool_messages 不污染会话历史
工具错误处理可以进一步标准化
可以给工具增加更细的参数 schema 和描述
```
