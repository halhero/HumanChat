# 16 调试模式与可观测性入口修改记录

## 本次目标

本次改造对应 `资料/11-框架化改造盘点.md` 中的第九项：可观测性与调试框架化。

本次先实现一个轻量的 CLI 调试入口，为后续接入 LangGraph streaming、节点事件和 LangSmith tracing 做准备。

## 修改前的问题

当前项目在运行时主要依靠：

```text
print
Python logging
异常提示
```

如果想知道一轮对话中：

```text
Graph 返回了哪些字段
是否发生了工具调用
有没有记忆候选
TTS 是否报错
```

需要手动改代码或查日志。

## 本次怎么改

在 CLI 中新增调试命令：

```text
/debug on
/debug off
```

开启后，每轮普通聊天结束会输出：

```text
Graph 返回字段
messages 数量
tool_messages 数量
memory_candidates 数量
tts_error
```

## 为什么这样做

这是一个低风险的可观测性入口。

它不会改变 Graph 执行逻辑，也不会引入额外服务，但能让开发者更容易观察当前状态流转。

后续如果接入：

```text
app.stream(...)
LangGraph 节点事件
LangSmith tracing
```

也可以继续沿用 `/debug on|off` 作为用户入口。
