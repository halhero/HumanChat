# 01 长期记忆闭环

## 目标

当前项目已经能读取 `data/memory/user_profile.json` 并注入系统提示词，但长期记忆还只是静态文件。本阶段要让长期记忆可查看、可手动维护，并为后续自动记忆提取预留入口。

## 设计目的

- 让用户能在 CLI 中直接查看长期记忆。
- 让用户能手动添加、删除 `preferences`、`facts`、`notes`。
- 保持长期记忆结构清晰，避免所有内容混在一个字符串里。
- 先实现人工可控的记忆管理，再考虑自动提取，降低“记错东西”的风险。

```text
/memory
/memory add preference 用户希望先看设计再改代码
/memory add fact HumanChat 使用 LangGraph
/memory delete preference 1
```

## 完成标准

- 输入 `/memory` 可以查看当前长期记忆。
- 输入 `/memory add ...` 可以写入长期记忆文件。
- 输入 `/memory delete ...` 可以删除指定记忆。
- 普通聊天流程不受影响。
