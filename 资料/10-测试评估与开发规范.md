# 10 测试、评估与开发规范

## 当前状态

HumanChat 已经有不少功能模块：

```text
memory_store
session_store
tools
runtime
graph
tts
stt
character
```

但目前缺少测试。

这意味着后续继续加功能时，很容易出现：

```text
改 memory 影响 graph
改 cli 影响 session 保存
改 tools 造成路径越权
升级依赖后某个模块导入失败
```

## 要做什么

本阶段要补上基础测试、最小评估和开发规范。

包括：

```text
引入 pytest
测试 memory_store
测试 session_store
测试 tools 路径安全
测试 runtime 保存逻辑
测试 Graph 在 mock LLM 下的基本流程
整理依赖管理
增加开发命令说明
```

这一步不是为了追求形式，而是为了让项目后续敢继续重构。

## 为什么这么做

成熟 Agent 项目的难点不只是“能跑”，而是“持续改也不容易坏”。

Agent 项目通常包含：

```text
模型调用
状态流转
外部工具
用户数据
音频能力
```

这些部分互相影响。如果没有测试，每一次修改都要靠人工跑完整流程，成本会越来越高。

测试的目标不是覆盖所有模型输出，而是保证确定性的本地逻辑可靠：

```text
记忆增删
JSON 保存恢复
工具安全边界
命令解析
运行层状态更新
```

## 怎么做

新增：

```text
tests/
  test_memory_store.py
  test_session_store.py
  test_tools.py
  test_runtime.py
```

示例：

```python
def test_add_memory_item_deduplicates():
    memory = LongTermMemory()
    assert add_memory_item(memory, "preference", "喜欢中文")
    assert not add_memory_item(memory, "preference", "喜欢中文")
```

工具安全测试：

```python
def test_read_project_file_blocks_outside_paths(tmp_path):
    with pytest.raises(ValueError):
        read_project_file(tmp_path, "../secret.txt")
```

Runtime 测试可以用 fake graph 或 monkeypatch：

```python
def test_runtime_saves_messages(...):
    ...
```

依赖管理方面，可以先保持 `requirements.txt`，但建议逐步增加：

```text
requirements-dev.txt
```

里面放：

```text
pytest
```

README 中补充：

```powershell
pip install -r requirements.txt
pip install -r requirements-dev.txt
python -m pytest
```

## 完成标准

这一阶段完成后，应具备：

```text
至少覆盖 memory/session/tools 的单元测试
工具路径安全有测试保护
运行层关键行为有测试
README 有测试运行说明
后续每次修改前后都能运行 python -m pytest
```
